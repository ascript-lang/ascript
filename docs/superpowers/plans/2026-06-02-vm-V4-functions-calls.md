# VM Plan V4 ‚Äî Functions & calls: FnProto/CLOSURE, CALL/RETURN, params/rest, multi-frame loop, call_value bridge

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development.

**Goal:** Compile function declarations and arrows to nested `FnProto`s and `CLOSURE`/`CALL`/`RETURN`; run multi-frame programs (frame push/pop on the explicit Fiber stack ‚Äî heap-bounded recursion); pass positional + rest params; and build the **load-bearing `call_value` native‚ÜîVM bridge** so native higher-order stdlib fns (`map`/`filter`/comparators/`recover`) can re-enter the VM. This is the slice that makes the VM Turing-complete for sync code and unlocks the whole stdlib.

**Architecture:** Add `Value::Closure(Rc<Closure>)` to `src/value.rs` (the one planned value-model addition besides V11's `shape_id`). The compiler emits a nested `FnProto` per function/arrow and a `CLOSURE proto-idx` to instantiate it (captures wired in V5; this plan: zero-capture or global-only functions first, then V5 adds upvalues). `CALL argc` pushes a `CallFrame`; `RETURN` pops it, leaving the return value. The VM's `run` loop already iterates frames; `CALL`/`RETURN` manipulate `fiber.frames`. The native bridge: `Vm::call_value(&self, callee, args) -> Result<Value,Control>` that, for a `Value::Closure`, spins a sub-run on the same Fiber (or a fresh frame) to completion; for `Builtin`/`Function`(legacy)/`Native`/etc. delegates to the borrowed `Interp::call_value`. **Depends on V3.**

---

## Ground truth
- `call_value` (interp): `async fn call_value(&self, callee:Value, args:Vec<Value>, span:Span) -> Result<Value,Control>` (`src/interp.rs:2080`) dispatches Builtin/Function/Class/BoundMethod/NativeMethod/GeneratorMethod/ClassMethod. The VM's `call_value` must handle `Value::Closure` (new) itself and delegate everything else to the interp's `call_value` (so legacy `Function`, builtins, native methods, classes all work during the alongside period).
- Higher-order stdlib (`array.map`, comparators, `recover`) call back via the interp's `call_value`. For those to invoke a **VM closure**, the interp's `call_value` must learn the new `Value::Closure` variant ‚Üí delegate to `Vm::call_value`. Wire a hook: the `Interp` holds an `Option<Weak<Vm>>` (or the Vm registers itself) so `interp.call_value(Closure,..)` routes back into the VM. Keep this bridge minimal and documented.
- Params: `Param { name, ty, rest }` (legacy) ‚Äî but the compiler reads params from the CST `ParamList`/`Param` nodes + resolver slots. Rest param collects trailing args into an array (`has_rest` fast-path, per CLAUDE.md). Arity/contract errors: surface like the tree-walker (Tier-2 panic with the same message); for async/gen they surface lazily (V7/V8).
- Type contracts on params/return: the tree-walker checks declared types at call (`run_body`). The VM's `CALL`/`RETURN` path must apply the SAME contract checks (reuse the interp's contract-check helper if callable; else replicate). Differential gate enforces identical panic messages.

## Tasks
- [ ] **T1 ‚Äî `Value::Closure` + arms.** Add `Closure(Rc<crate::vm::Closure>)` to `value.rs` `Value` (update equality = pointer identity; `Clone`; `Display`/`type` name = "function" to match tree-walker's function type string; any exhaustive matches in value.rs/stdlib that must handle it ‚Äî search and add arms, defaulting to the function-like behavior). Gate: full suite still green (the new variant is inert until produced). Commit `feat(value): Value::Closure variant`.
- [ ] **T2 ‚Äî compile functions to protos.** `FnDecl`/`ArrowExpr` ‚Üí build a child `FnProto` (own `Chunk`, arity, has_rest, is_async/is_generator flags from the CST) by recursively compiling the body with a fresh `Compiler` scope keyed to that function's resolver frame (`frames[(kind,range)]` ‚Üí slot_count/upvalues). Emit `add_proto` + `CLOSURE proto-idx` at the declaration site; bind the resulting closure to the fn's slot (FnDecl) or leave on stack (arrow expr). Disassembler shows nested protos. Test via disasm: `fn f(){return 1}` emits a proto with `CONST 1; RETURN` and a `CLOSURE` in the parent. Commit.
- [ ] **T3 ‚Äî CALL/RETURN + multi-frame.** `CallExpr` ‚Üí compile callee, then args left-to-right, then `CALL argc`. Exec `CALL`: peek callee under the args; if `Value::Closure`, push a `CallFrame { closure, ip:0, slot_base }` where `slot_base` places the args into the new frame's first slots (params), reserve the rest of `slot_count` as `Nil`; continue the loop in the new frame. `RETURN`: take top as result, pop the frame, truncate stack to `slot_base`, push result; if no frames remain ‚Üí `RunOutcome::Done(result)`. Recursion is heap-bounded (frames are heap `Vec`). Tests: `fn fac(n){ if n<2 {return 1} return n*fac(n-1) }\nprint(fac(10))` ‚Üí `3628800`; deep recursion (e.g. fac(5000) sum) doesn't overflow the Rust stack (a test the tree-walker might fail ‚Äî note it). Commit.
- [ ] **T4 ‚Äî params + rest + arg arity/contracts.** Bind positional args to param slots; a `...rest` param collects trailing args into a `Value::Array`. Apply declared param/return type contracts identically to the tree-walker (reuse `Interp`'s contract helper). Arity mismatch ‚Üí identical Tier-2 panic. Tests: rest params, typed params (`fn f(n:number)` called with a string ‚Üí identical panic), wrong arity. Commit.
- [ ] **T5 ‚Äî `call_value` bridge (native‚ÜîVM).** Implement `Vm::call_value(&self, callee, args) -> Result<Value,Control>`: `Value::Closure` ‚Üí run it on a fresh Fiber (or a nested frame) to `Done`; everything else ‚Üí `self.interp.call_value(callee, args, span)`. Then wire the REVERSE: teach `Interp::call_value` to recognize `Value::Closure` and route to the registered `Vm` (install a `Weak<Vm>` on the interp at VM construction). Now `array.map(closure)`, comparators, and `recover(closure)` work. Tests: `[1,2,3].map(x => x*2)` via VM == tree-walker; `recover(() => 1)`; a comparator sort. Commit.
- [ ] **T6 ‚Äî widen differential gate.** Add function-heavy sync examples (`functions.as`, `factorial.as`, parts of `stdlib.as` using map/filter) to the allow-list. Byte-identical. Full suite + clippy both configs. Commit.
