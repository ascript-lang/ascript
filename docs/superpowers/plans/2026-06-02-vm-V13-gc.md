# VM Plan V13 ‚Äî Cycle-collecting GC: Rc‚ÜíCc migration + Bacon‚ÄìRajan trial deletion (closing phase)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development.
> **The FINAL phase of the VM sub-project.** Done AFTER the VM reaches whole-corpus behavioral parity on `Rc` (V10) and the perf/persistence layers (V11‚ÄìV12), over the VM's FINAL representation ‚Äî a single `Rc ‚Üí Cc` migration, no double-churn. The behavioral oracle (VM == tree-walker) and this MEMORY proof are kept in separate phases so a behavioral bug and a collector bug never share a phase.

**Goal:** Reclaim reference cycles in `Rc`-backed mutable data + closures (`let a=[]; a.push(a)`, mutually-referencing objects/instances, mutually-capturing closures, Fiber‚ÜîFuture/Generator loops) ‚Äî WITHOUT breaking deterministic native-resource cleanup. Use `gcmodule`'s `Cc<T>` (refcounting + Bacon‚ÄìRajan trial-deletion): acyclic objects keep immediate `Drop`; only actual cycles are collected.

**Architecture:** Migrate the cycle-capable types from `Rc`/`Rc<RefCell>` to `Cc`/`Cc<RefCell>` (gcmodule): `Array`, `Object`, `Map`, `Set` inner cells; `Instance`; upvalue cells (`Rc<RefCell<Value>>` ‚Üí `Cc<RefCell<Value>>`); `Closure` (captures other values); Fiber/Generator handles that can loop with `Future`. Implement `Trace` for all cycle-capable types (incl. `IndexMap`-backed `Object`/`Map`). With `Cc`, the operand stack/frames hold ordinary `Cc` strong refs and external references are accounted by the refcount ‚Äî NO separate root-enumeration API needed. Native-resource handles (`Native(Rc<NativeObject>)`, TCP/file/DB/child/cancel-on-drop tasks) stay on `Rc` (acyclic, deterministic `Drop`) OR are proven never trapped in a collected cycle. **Depends on V12.** This touches `value.rs` broadly ‚Äî do it once, over the final representation.

---

## Ground truth
- Cycle-capable (must migrate to `Cc` + `Trace`): `Array(Rc<RefCell<Vec<Value>>>)`, `Object(Rc<RefCell<IndexMap<String,Value>>>)`, `Map(Rc<RefCell<IndexMap<MapKey,Value>>>)`, `Set(Rc<RefCell<IndexSet<MapKey>>>)`, `Instance(Rc<RefCell<Instance>>)`, `Function`/`Closure` (capture values), upvalue cells, Fiber/Generator/Future loops.
- Acyclic / native (keep `Rc`, deterministic `Drop`): `Str(Rc<str>)`, `Builtin`, `Regex`, `Native(Rc<NativeObject>)` + its OS resources, `Class`/`EnumDef` (immutable defs ‚Äî but methods/def_env may hold values; assess). `SharedFuture` cancel-on-drop MUST keep its `Drop` timing.
- `gcmodule`: `Cc<T>` strong ref; `Trace` trait; `collect_thread_cycles()` (or equivalent) for collection; `!Send` (no atomics) ‚Äî fits the current-thread runtime.
- Soundness gates are the MEMORY proof, distinct from the behavioral oracle.
