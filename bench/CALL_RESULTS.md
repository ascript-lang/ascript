# Call-Path Allocation Diet + Callback Trampoline Benchmark (CALL)

Tracks the same-session A/B measurements for the CALL spec
(`superpowers/specs/2026-06-12-call-path-diet-design.md`, plan
`superpowers/plans/2026-06-12-call-path-diet.md`).

**Workloads:**
- `func_pipeline` — map/filter/reduce pipelines (the headline CALL target)
- `call_heavy` — tight function-call loop (the secondary CALL target)
- `object_churn` — object create + index/member reads (regression guard)
- `json_roundtrip` — encode + decode (regression guard)
- `server_request` — single-isolate HTTP-server requests (async regression guard)

**Goal:** CALL reduces per-call allocations (A1/A2/A3) and per-element callback
ceremony (Unit B / trampoline), measurably improving `func_pipeline` and `call_heavy`
while leaving the non-functional workloads within noise.

### A1

**Change:** `alloc_cells` returns `Vec::new()` when `cell_slots` is empty (the
common case — most functions capture nothing by reference). Every binding loop
switched from `cells[slot]` (panics on empty vec) to `cells.get(slot).and_then(|c| c.as_ref())`
(bounds-tolerant; behavior-identical). A1 is always-on (not gated on `call_fast`
since it is behavior-invisible). Saves ~1 heap alloc per capture-free call.

**Alloc slope (debug, N=20k):** pre-A1 ~3.0/call → post-A1 ~2.0/call (cells-vec gone).

### A2

**Change:** In-place argument binding over the operand-stack window (CALL §3).
The qualifying `Op::Call` arm (`call_fast=true`, `!has_rest`) now runs
`check_call_args_in_place` (borrows the stack window, no `Vec`) then does
`fiber.stack.remove(callee_idx)` (an argc-element memmove) + `resize` to fill
defaults with `Nil`. Eliminates 2 heap allocations per qualifying call: the
`vec![Value::Nil; argc]` in `push_closure_frame` and the `BoundArgs.values` Vec
from `check_call_args`. Combined with A1 (cells-vec gone), the qualifying call
shape (capture-free, no rest params) now allocates **0** per call in the fast path.

The shared arity + contract logic is extracted into `check_call_arity` +
`check_param_contract` cores consumed by both `check_call_args` and the new
`check_call_args_in_place` — wording is byte-identical by construction (CALL §3.3).

**Alloc slope (debug, N=20k):**
- `call_fast=true` (A1+A2): below 1.0/call (2 arg Vecs eliminated; gate passed)
- `call_fast=false` (A1 only): below 3.0/call (cells-vec gone, 2 arg Vecs remain)

**vm_differential:** 424 passed, 0 failed (both feature configs).
