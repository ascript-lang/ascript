## 1. Structural fact — `size_of::<Value>()`

| | bytes |
|---|---|
| pre-VAL baseline (@ `612339c`) | **32** |
| VAL Stage 1 (this branch) | **24** |

The honest Stage-1 floor is **24**, not 16: the inline scalar variants
(`Int`/`Float`) take any bit pattern, so Rust cannot niche-elide the
discriminant — the layout is `round_up(widest_payload) + 8-byte tag`. With
the fat `Str(Rc<str>)` (16-byte 2-word pointer) still the widest payload,
that is 16 + 8 = 24. Reaching 16 needs thin-`Str` (Task 9); 8 needs the
NaN-box (Stage 2, gated). Boxing the two fat method-binding variants
(`ClassMethod`/`GeneratorMethod`, 24-byte payloads) was the load-bearing
32→24 shrink; boxing `Decimal` removed the other 16-byte inline payload.

## 3. Geomean (HOT workloads) — VAL vs same-session baseline

| Mode | baseline geomean (ms) | VAL geomean (ms) | speedup | delta |
|------|-----------------------|------------------|---------|-------|
| **specialized** | 2970.0 | 2957.3 | 1.004× | +0.4% |
| **generic (`--no-specialize`)** | 3068.7 | 3064.7 | 1.001× | +0.1% |

**Gate 12: PASS.** VAL's win is an encoding optimization UNDER the value
API (not a specialization guard), so the generic VM — which skips every
IC/adaptive/global fast path — must benefit from the smaller `Value` too and
must NOT regress. A generic-mode regression would be a VAL bug, not an
acceptable trade.

**Honest reading:** on this host (a fast Apple-silicon core) these workloads
are CPU-bound, not memory-bandwidth-bound, so the 32→24 shrink lands at the
**noise floor** — both modes' geomeans sit within ±1% of the baseline across
runs (a positive delta on one run, a slightly negative one on the next). No
speedup is claimed; the load-bearing facts are (1) the structural 32→24
shrink and (2) **no regression in EITHER mode** beyond noise (Gate 12). The
cache-density win this shrink buys becomes measurable on memory-bound,
large-`Vec<Value>`/`IndexMap`-traversal workloads and compounds with the
later stages (thin-`Str`→16, NaN-box→8); Stage 1 alone is a correctness +
foundation step, not a throughput headline.

## 4. Cold-path check (Task 1 / Task 2 boxing)

| Workload | base spec (ms) | VAL spec (ms) | delta | notes |
|----------|----------------|---------------|-------|-------|
| decimal_cold | 142.6 | 159.0 | -10.3% | Decimal boxed to `Rc<Decimal>` — one `Rc::new` per op |
| method_cold | 755.7 | 747.5 | +1.1% | `ClassMethod`/`GeneratorMethod` boxed to one `Rc` payload |

Honest framing: boxing `Decimal` means decimal arithmetic now does an
`Rc::new` allocation per op (the cold path). On any NON-decimal workload this
code never runs, so it adds zero to the hot path; `decimal_cold` measures the
cold cost directly. The two method-binding variants are rare, cold bindings —
`method_cold` confirms the extra indirection on their construct+dispatch is
not a measurable regression.
