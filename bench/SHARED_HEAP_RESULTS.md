## 1. Headline — shared-heap zero-copy vs per-dispatch deep-clone

Per-dispatch cost of handing a table of N entries to a worker isolate,
two ways: **clone** = pass the plain table (the airlock deep-copies it,
O(N)); **shared** = `shared.freeze` once then pass the `Value::Shared`
(the airlock bumps an `Arc`, O(1)). The worker returns a tiny scalar, so
the time is dominated by argument transport.

| Table size (entries) | Deep-clone / call (ms) | Shared / call (ms) | Speedup |
|----------------------|------------------------|--------------------|---------|
| 5,000 | 0.619 | 0.155 | **4×** |
| 10,000 | 1.059 | 0.144 | **7×** |
| 25,000 | 2.392 | 0.160 | **15×** |
| 50,000 | 4.729 | 0.150 | **31×** |

The shared per-call time is **flat** (one atomic increment regardless of
table size); the deep-clone time grows **linearly** — so the advantage
grows with the table. This is the per-request shared-config win: freeze a
routing table / feature-flag snapshot once, read it across N isolates, pay
zero per-dispatch copy.

### Freeze cost (one-time, amortized)

| Table size (entries) | Freeze (ms) |
|----------------------|-------------|
| 5,000 | 0.22 |
| 10,000 | 0.52 |
| 25,000 | 1.42 |
| 50,000 | 3.01 |

Freeze is O(distinct nodes), paid ONCE; every subsequent dispatch is O(1).

## 2. Gate 12 — no per-read tax on the non-shared hot path

The `Value::Shared` read arm is a single tag check AFTER the existing
Object/Array/Map IC fast paths, so an ordinary (non-frozen) receiver never
reaches it (`ic_get_field` returns `None` for a `Shared` receiver and the
specialized path falls through to the generic reader). `object_churn.as`
(6M object create + index/member read) is the dispatch-bound hot path; its
steady state must not regress.

**Same-session A/B control** (the only honest way to read a ±noise delta).
Both binaries built `--profile profiling` in the same session; the pre-SRV
baseline is `main` @ 4021d42 (FFI merge, the commit SRV branched from) built
in an isolated worktree. Runs are **interleaved** (branch, main, branch, …)
so machine drift cancels:

| round | branch (SRV) ms | main (pre-SRV) ms |
|-------|-----------------|-------------------|
| 1 | 4823 | 4790 |
| 2 | 4795 | 4806 |
| 3 | 4808 | 4767 |
| 4 | 4800 | 4799 |
| 5 | 4778 | 4801 |
| 6 | 4775 | 4790 |
| **median** | **4797** | **4794** |

Delta vs same-session baseline: **+0.1%** → **PASS**. The two series
interleave (several branch runs beat the corresponding main run), so the
difference is pure run-to-run noise, not a regression — exactly what the
"added arm is unreachable on the non-Shared hot path" architecture predicts.
(An earlier draft compared against a baseline recorded on a different day /
build and reported a misleading +11.5%; replaced with this controlled A/B.)
The three-way `vm_differential` gate separately proves specialized == generic
byte-identically over the corpus, including the shared examples.

*Headline curve generated by `bench/run_shared_heap_bench.sh`; the Gate-12
A/B above is a same-session control (see the section body for method).*
